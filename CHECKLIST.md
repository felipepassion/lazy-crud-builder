## ✅ Template Structure Compliance

All templates follow the established pattern:

### Include Structure
- [x] All templates include `Tests.Common.ttinclude` (not `project.tt` directly)
- [x] `Tests.Common.ttinclude` provides shared helpers
- [x] Proper relative path: `<#@ include file="..\..\ProjectFiles\Tests\Tests.Common.ttinclude" #>`

### Output Generation
- [x] Use `SaveOutputToSubFolder()` for consistent output
- [x] Generate to `GetTestOutputPath()` subfolder
- [x] File naming: `{AggName}.{TestClassName}.g.cs`

### Code Headers
- [x] All generated files include `// <auto-generated>` header
- [x] All include XML summary: `/// <summary>Auto-generated by LazyCrud. DO NOT EDIT.</summary>`

### Test Framework
- [x] All use xUnit framework
- [x] Use `[Fact]` for single tests
- [x] Use `[Theory]` with `[InlineData]` for parameterized tests
