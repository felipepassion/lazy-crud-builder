# E2E Local Testing for LazyCrudBuilder

Este diretório contém scripts para automação do fluxo E2E (end-to-end) do LazyCrudBuilder.

## Fluxo E2E - Products Aggregate

O script `e2e-products.ps1` automatiza o seguinte fluxo:

### 1. Configuração do lazy.settings
- Define `TemplatesSource = Local`
- Define `UseNugetTemplates = False`
- Aponta para `src/ProjectFiles` como source de templates locais

### 2. Delete do Aggregate Products
- Remove completamente o diretório `src/Products`
- Remove todas as referências a Products projetos da solution
- Reset completo do aggregate

### 3. Recriar Products do Zero
Cria a estrutura de camadas completa:
- **Domain**: `Products.Domain`
- **Enumerations**: `Products.Enumerations`
- **Application DTO**: `Products.Application.DTO`
- **Application**: `Products.Application`
- **Infrastructure Data**: `Products.Infra.Data`
- **Infrastructure IoC**: `Products.Infra.IoC`
- **API**: `Products.Api`
- **API Queries**: `Products.Api.Queries`
- **Tests**: Todos os projetos de teste correspondentes

### 4. Criar T4 Wrappers
Cria os arquivos `.tt` wrapper em cada camada que apontam para os templates locais em `src/ProjectFiles`:

**Camadas de Produção:**
- Domain → `..\..\ProjectFiles\Domain\Domain.tt`
- Enumerations → `..\..\ProjectFiles\Enumerations\Enumerations.tt`
- etc.

**Camadas de Teste:**
- Tests.Domain → `..\..\ProjectFiles\Tests\Domain\Domain.Tests.tt`
- Tests.DTO → `..\..\ProjectFiles\Tests\Application.DTO\DTO.Tests.tt`
- Tests.Application → `..\..\ProjectFiles\Tests\Application\Application.Tests.tt`
- etc.

### 5. Build e Test
- `dotnet restore`
- `dotnet build`
- `dotnet test`

## Como Usar

### Execução Básica
```powershell
.\build\e2e-products.ps1
```

### Com Clean Completo
```powershell
.\build\e2e-products.ps1 -Clean
```

## Requisitos

1. **PowerShell 7+** (pwsh)
2. **.NET SDK** instalado
3. **Visual Studio** (para executar T4 templates) ou `msbuild` com T4 support

## Ordem de Execução de T4 Templates

Quando rodar os T4 templates (manualmente ou via automation), seguir esta ordem:

1. **Domain** - Entidades base, repositories, specifications
2. **Enumerations** - Enums e valores
3. **Application.DTO** - DTOs, requests, responses
4. **Application** - AppServices, command handlers
5. **Infra.Data** - DbContext, mappings, repositories
6. **Infra.IoC** - Dependency injection
7. **Api** - Controllers, endpoints
8. **Api.Queries** - Query models
9. **Tests** - Todos os testes (rodar por último)
   - Tests.Domain
   - Tests.Enumerations
   - Tests.DTO
   - Tests.Application
   - Tests.InfraData
   - Tests.InfraIoC
   - Tests.Api
   - Tests.ApiQueries

## Idempotência

O script é **idempotente** - pode ser executado múltiplas vezes:
- Se Products já existe → deleta e recria
- Se Products não existe → cria do zero
- Sempre sobrescreve `lazy.settings` com valores corretos
- Não falha se diretórios já existem

## Troubleshooting

### Erro: "lazy.settings not found"
- Verifique que está executando do diretório correto
- Path esperado: `src/Mods/lazy.settings`

### Erro: "dotnet build failed"
- Execute `dotnet restore` primeiro
- Verifique se todos os projetos têm .csproj válidos
- Rode T4 templates antes do build

### Testes falhando
- Execute os T4 templates em ordem (ver seção acima)
- Verifique se arquivos `.generated.cs` foram criados
- Valide que o template aponta para o ProjectFiles correto

### T4 não encontrado
- Instale Visual Studio com workload de desenvolvimento .NET
- Ou use `dotnet-t4` package
- Ou execute via `msbuild /t:TransformAll`

## Contribuindo

Para adicionar novos aggregates ao E2E:
1. Copie `e2e-products.ps1`
2. Renomeie para `e2e-<aggregate>.ps1`
3. Substitua "Products" pelo nome do novo aggregate
4. Ajuste a estrutura de camadas se necessário
