# Test Templates for LazyCrudBuilder

Auto-generated test templates for xUnit. These templates generate test files per entity/aggregate with the following structure:

## Structure

```
Tests/
├── Tests.Common.ttinclude          # Shared helpers for all test templates
├── Domain/
│   └── Tests.Domain.tt             # Entity tests (ctor, ToString, properties)
├── Application/
│   └── Tests.Application.tt        # AppService interface tests
├── Application.DTO/
│   └── Tests.DTO.tt                # DTO serialization tests
├── Infra.Data/
│   └── Tests.InfraData.tt          # Repository & mapping tests
├── Infra.IoC/
│   └── Tests.InfraIoC.tt           # DI registration tests
├── Api/
│   └── Tests.Api.tt                # Controller & endpoint tests
├── Api.Queries/
│   └── Tests.ApiQueries.tt         # QueryModel tests
└── Enumerations/
    └── Tests.Enumerations.tt       # Enum parsing & value tests
```

## Output

All generated test files include:

- `// <auto-generated>` header
- `/// <summary>Auto-generated by LazyCrud. DO NOT EDIT.</summary>` documentation
- xUnit test classes with `[Fact]` and `[Theory]` tests

## Test Coverage by Layer

### Domain (Tests.Domain.tt)
- Entity instantiation
- Default property values
- ToString behavior
- Property setters (for strings, nullable types, value types)

### Application.DTO (Tests.DTO.tt)
- DTO instantiation
- JSON serialization/deserialization with `System.Text.Json`
- Round-trip serialization tests

### Application (Tests.Application.tt)
- AppService interface existence
- Interface method discovery

### Infra.Data (Tests.InfraData.tt)
- Repository interface existence and type validation
- Entity mapping class existence

### Infra.IoC (Tests.InfraIoC.tt)
- IoCFactory class existence
- RegisterDependencies/Configure method discovery
- Repository and AppService type resolution

### Api (Tests.Api.tt)
- Controller class existence
- Controller inheritance from ControllerBase
- Route attribute presence
- HTTP method attributes (Get, Post, Put, Delete) based on `EndpointsT4` attributes

### Api.Queries (Tests.ApiQueries.tt)
- QueryModel instantiation
- Default pagination support
- JSON serialization
- Filter properties
- OrderBy and pagination configuration

### Enumerations (Tests.Enumerations.tt)
- Enum member discovery
- String parsing (`Enum.Parse`, `Enum.TryParse`)
- ToString behavior
- Valid/invalid member parsing

## Usage in LazyCrud

These templates are packaged in NuGet and referenced by the consuming LzyCrud generator. The consumer creates wrapper `.tt` files that:

1. Include the appropriate test template
2. Set aggregate and namespace context
3. Generate output to the desired test project location

## Helpers (Tests.Common.ttinclude)

Common functions available to all test templates:

- `WriteAutoGeneratedHeader(layerName, entityName)` — Standard header
- `NamespaceForTests(layerName, aggName)` — Builds test namespace
- `NormalizeTestClassName(entityName, suffix)` — Entity test class name
- `NormalizeDtoTestClassName(entityName)` — DTO test class name
- `NormalizeControllerTestClassName(entityName)` — Controller test class name
- `NormalizeQueryModelTestClassName(entityName)` — QueryModel test class name
- `GetTestOutputPath()` — Test output subfolder path
- `GetTestOutputPathForLayer(layerName)` — Layer-specific output path
- `HasEndpoints(entity)` — Check if entity has API endpoints

## Notes

- The Builder only maintains and packages templates
- LzyCrud organizes and generates output/test projects per aggregate
- Templates follow the same patterns as existing production templates
- All generated files include auto-generated headers to prevent manual editing
